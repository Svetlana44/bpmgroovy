# Дополнительные вопросы для подготовки

## QA Automation / Framework

- Как организовать теги и запуски тестов в CI?  
  Ответ: разделить тесты по тегам и запускать отдельными jobs (api/ui/contract),
  хранить отчёты как артефакты, запускать быстрые тесты на каждый MR.

- Как уменьшить флейки UI тестов без глобальных sleep?  
  Ответ: стабильные локаторы, явные ожидания, изоляция данных, фиксация времени,
  отказ от sleep, только ожидания по условиям.

- Как строить тест‑данные при параллельном запуске?  
  Ответ: уникальные идентификаторы (UUID), отдельные тестовые пользователи,
  изоляция по namespace/tenant, teardown после тестов.

## API/Contract

- Что такое idempotency‑key и где применяется?  
  Ответ: ключ, который делает повтор запроса безопасным (например, платежи).
  Сервер не выполняет операцию повторно.

- Как валидировать OpenAPI спецификацию в CI?  
  Ответ: хранить спецификацию в репо, прогонять schema‑validation в тестах
  и фейлить pipeline при нарушении контракта.

- Разница между contract‑tests и integration‑tests?  
  Ответ: contract проверяет формат взаимодействия, integration проверяет
  реальное взаимодействие компонентов и окружения.

## Java

- Как работает HashMap (коллизии, tree bins)?  
  Ответ: ключи хэшируются, коллизии складываются в bucket, при высокой
  коллизии bucket становится деревом (tree bin).

- Что такое immutable коллекции?  
  Ответ: неизменяемые коллекции, защищают от побочных эффектов и потоковых ошибок.

- Разница между ArrayList и LinkedList по сложностям?  
  Ответ: ArrayList быстрый доступ по индексу O(1), вставки в середину O(n);
  LinkedList — доступ O(n), вставки в середину O(1) при наличии узла.

## БД

- Что такое ACID?  
  Ответ: атомарность, согласованность, изоляция, долговечность.

- Когда индекс может ухудшить производительность?  
  Ответ: при частых INSERT/UPDATE/DELETE, из‑за обновления индекса.

- Что такое транзакции и уровни изоляции?  
  Ответ: группа операций как единое целое, уровни изоляции регулируют
  видимость данных (read committed, repeatable read, serializable).
