# Аутентификация и безопасность API

## Вопросы и ответы

**1. Чем отличается аутентификация от авторизации?**  
Аутентификация — подтверждение личности. Авторизация — проверка прав.

**2. Какие схемы аутентификации чаще всего встречаются?**  
Basic Auth, Bearer Token (JWT), API Key, OAuth 2.0, OpenID Connect.

**3. Что проверять в API тестах безопасности?**  
- отсутствие доступа без токена  
- доступ по ролям  
- срок жизни токена  
- корректность refresh‑flow  
- инъекции (SQL/XSS)  
- rate limiting

**4. JWT: что внутри и как проверять?**  
Header, Payload, Signature. Проверка сроков (`exp`), ролей, issuer/audience.

**5. Как тестировать RBAC/ABAC?**  
Набор ролей/атрибутов, матрица доступов, негативные кейсы.

**6. Что такое rate limiting?**  
Ограничение частоты запросов. Тесты — burst‑нагрузка, ожидание 429, retry.

**7. Какие заголовки безопасности должны быть?**  
HSTS, Content‑Security‑Policy, X‑Content‑Type‑Options, X‑Frame‑Options.

**8. Типичный сценарий теста авторизации?**  
```
1) Без токена -> 401
2) С токеном без роли -> 403
3) С токеном с ролью -> 200
```

**9. Что важно для Senior‑уровня?**  
Понимание угроз, негативные сценарии, политика доступа, совместная работа
с DevOps и Security.
