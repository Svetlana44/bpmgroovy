# Тестирование REST API

## Вопросы и ответы

**1. Что такое REST и его ключевые принципы?**  
Ресурсная модель, единый интерфейс, stateless, кеширование, layered system.

**2. Какие уровни тестов применимы к API?**  
Unit (контроллер/сервис), интеграционные (с БД), контрактные, E2E.

**3. Что нужно проверять в API тестах?**  
- статус‑коды  
- структура и типы данных  
- обязательные поля  
- бизнес‑валидации  
- корректность ошибок  
- идемпотентность

**4. Как строить негативные тесты?**  
Плохая схема запроса, невалидные типы, пустые поля, отсутствие авторизации,
несуществующие id, превышение лимитов.

**5. Как проверять JSON Schema?**  
Схему хранить в ресурсах и валидировать ответ.

```java
given()
.when()
    .get("/api/users/123")
.then()
    .statusCode(200)
    .body(matchesJsonSchemaInClasspath("schemas/user.json"));
```

**6. Что такое идемпотентность и как тестировать?**  
Повтор запроса не меняет результат. Пример: `PUT`/`DELETE`.

**7. Как тестировать пагинацию?**  
Проверка `page/size`, сортировки, корректности метаданных и стабильности.

**8. Какие инструменты ты используешь?**  
RestAssured, Postman, Wiremock, Pact, OpenAPI Validator.

**9. Что такое contract‑first?**  
API сначала описывается в OpenAPI, затем реализация и тесты валидируются
относительно контракта.

**10. Что важно показать на Senior‑уровне?**  
Стратегия тестирования, изоляция, стабильность тестов, анализ дефектов,
качественные метрики и отчётность.
