## Типичные вопросы и ответы

### 1. Что такое контрактное тестирование?

**Правильный ответ:**
"Контрактное тестирование - это подход к тестированию, при котором проверяется совместимость между сервисами через контракты (API контракты), без необходимости запуска реальных сервисов.

Основные преимущества:
- Быстрые тесты (не требуют запуска всех сервисов)
- Раннее обнаружение несовместимостей
- Независимость от внешних зависимостей

Инструменты: OpenAPI/Swagger"

**Что показать:**
- Понимание концепции Consumer/Provider
- Знание инструментов (OpenAPI/Swagger)
- Практический опыт (если есть)

### 2. Как работает Wiremock?

**Правильный ответ:**
"Wiremock - это инструмент для мокирования HTTP сервисов. Он позволяет:
- Создавать заглушки (stubs) для внешних API
- Изолировать тесты от внешних зависимостей
- Тестировать различные сценарии (успех, ошибки, таймауты)
- Записывать и воспроизводить запросы

Используется для:
- Изоляции тестов
- Тестирования интеграций без реальных сервисов
- Создания тестовых данных"

**Что показать:**
- Пример настройки Wiremock
- Пример создания заглушки
- Понимание, когда использовать

См. пример: `../examples/contract/WiremockExampleTest.java`.

### 3. Разница между Selenium и Playwright?

**Правильный ответ:**
"Основные различия:

**Selenium:**
- Стандарт индустрии, много ресурсов
- Требует явных ожиданий (WebDriverWait)
- Медленнее Playwright
- Поддержка многих языков

**Playwright:**
- Современный инструмент от Microsoft
- Автоматические ожидания элементов
- Быстрее Selenium
- Встроенные скриншоты и видео
- Лучшая поддержка современных веб-приложений

Выбор зависит от проекта, но Playwright часто предпочтительнее для новых проектов."

См. пример: `../examples/ui/PlaywrightExampleTest.java`.

### 4. Как тестировать API?

**Правильный ответ:**
"Подходы к тестированию API:

1. **Функциональное тестирование:**
   - Проверка статус кодов
   - Валидация структуры ответа
   - Проверка бизнес-логики

2. **Негативное тестирование:**
   - Невалидные данные
   - Отсутствующие обязательные поля
   - Неправильные типы данных

3. **Безопасность:**
   - Аутентификация/авторизация
   - SQL injection
   - XSS

4. **Производительность:**
   - Время ответа
   - Нагрузочное тестирование

Инструменты: RestAssured, Postman, SoapUI"

### 5. Какие паттерны используются в автотестах?

**Правильный ответ:**
"Основные паттерны:

1. **Page Object Model (POM):**
   - Инкапсуляция элементов страницы
   - Переиспользование кода
   - Упрощение поддержки

2. **Builder Pattern:**
   - Создание сложных объектов
   - Улучшение читаемости тестов

3. **Factory Pattern:**
   - Создание объектов тестовых данных
   - Централизованное управление

4. **Singleton:**
   - Утилитные классы (например, PropertiesReader)
   - Единая точка доступа

5. **Strategy:**
   - Разные стратегии тестирования
   - Выбор алгоритма во время выполнения"

### 7. В чем разница между MongoDB и Redis?

**Правильный ответ:**
"MongoDB и Redis - это разные типы NoSQL баз данных:

**MongoDB:**
- Документо-ориентированная БД
- Хранит данные на диске
- Поддерживает сложные запросы и агрегации
- Используется для хранения документов, каталогов, логов

**Redis:**
- In-memory key-value хранилище
- Хранит данные в RAM (очень быстро)
- Ограниченные возможности запросов
- Используется для кэширования, сессий, очередей

В тестах: MongoDB - для проверки данных, Redis - для кэширования и моков."

**Что показать:**

- Примеры применения каждого
- См. также: `../comparisons/MongoDB_vs_Redis.md`

## Практические задания

### Задание 1: Написать API тест

**Типичное задание:**
"Напишите тест для API `/api/users/{id}`, который:
- Проверяет успешный ответ (200)
- Валидирует структуру JSON
- Проверяет негативные сценарии (404, 400)"

**Решение:**
```java
@Test
void testGetUser() {
    given()
        .pathParam("id", 1)
    .when()
        .get("/api/users/{id}")
    .then()
        .statusCode(200)
        .body("id", equalTo(1))
        .body("name", notNullValue())
        .body("email", matchesPattern(".*@.*\\..*"));
}

@Test
void testGetUserNotFound() {
    given()
        .pathParam("id", 999)
    .when()
        .get("/api/users/{id}")
    .then()
        .statusCode(404);
}
```

### Задание 2: Написать UI тест

**Типичное задание:**
"Напишите тест для формы логина:
- Заполнить поля
- Нажать кнопку
- Проверить результат"

**Решение (Selenium):**
```java
@Test
void testLogin() {
    driver.get("https://example.com/login");
    
    driver.findElement(By.id("username")).sendKeys("user");
    driver.findElement(By.id("password")).sendKeys("pass");
    driver.findElement(By.id("submit")).click();
    
    WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    WebElement success = wait.until(
        ExpectedConditions.visibilityOfElementLocated(By.id("success"))
    );
    
    assertTrue(success.isDisplayed());
}
```

### Задание 3: SQL запрос

**Типичное задание:**
"Напишите SQL запрос, который вернет пользователей с количеством их заказов"

**Решение:**
```sql
SELECT 
    users.name,
    users.email,
    COUNT(orders.id) as order_count
FROM users
LEFT JOIN orders ON users.id = orders.user_id
GROUP BY users.id, users.name, users.email
ORDER BY order_count DESC;
```
