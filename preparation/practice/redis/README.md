# Практика: Redis

## Цели
- Понять основные структуры данных Redis
- Проверить сценарии кэширования и TTL
- Научиться чистить данные между тестами

## Минимальный стенд
- Локальный Redis или Docker
- Клиент Jedis/Lettuce для Java

## Упражнения
1. Записать и прочитать ключи типа String.
2. Создать Hash с данными пользователя и проверить чтение.
3. Проверить TTL: записать ключ с `expire` и убедиться, что он исчезает.
4. Смоделировать кэш: записать ответ API и проверить повторное чтение.
5. Очистить namespace тестов через `DEL` по маске (или отдельную БД).

## Подсказки
- Используй префикс `test:` для всех ключей.
- Для изоляции можно использовать отдельную базу Redis.
