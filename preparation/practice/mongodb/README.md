# Практика: MongoDB

## Цели
- Научиться выполнять базовые CRUD операции
- Освоить фильтрацию и работу с индексами
- Понимать, как проверять данные после API/интеграционных тестов

## Минимальный стенд
- Локальная MongoDB или Docker
- Тестовая база и коллекция `users`

## Упражнения
1. Создать коллекцию `users` и добавить 5 документов с разной структурой полей.
2. Выполнить CRUD операции и проверить, что данные сохранились.
3. Написать запрос с фильтром по вложенному полю.
4. Добавить индекс и сравнить время запроса (на примере explain).
5. Реализовать очистку тестовых данных (по метке или email).

## Данные для примеров
Используй поля: `name`, `email`, `roles`, `address.city`, `createdAt`.

## Подсказки
- Храни тестовые данные с уникальным префиксом, чтобы легко чистить.
- Фиксируй ожидаемое состояние через выборку `find` после теста.
