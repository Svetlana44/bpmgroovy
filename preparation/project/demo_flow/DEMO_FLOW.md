# Demo‑flow для интервью (Senior QA Automation Java)

## Цели демонстрации

- Показать архитектурное мышление: уровни тестов, слои фреймворка
- Показать стабильность: устойчивые UI/API тесты, надежные ожидания
- Показать качество: контракты, схемы, отчётность, CI/CD
- Показать зрелость: стратегия, метрики, работа с дефектами

## Сценарий демонстрации (15–20 минут)

1. **Краткий обзор репозитория**
   - структура `src/test/java` (API/UI/contract)
   - конфигурация сборки в `build.gradle`

2. **API‑тесты**
   - RestAssured тест, JSON‑Schema валидация
   - пример негативного сценария

3. **Контрактные тесты**
   - кратко: OpenAPI validation vs Pact
   - где лежат примеры

4. **UI‑тесты**
   - Page Object, ожидания, скриншоты
   - сравнение Selenium/Selenide/Playwright (когда что)

5. **Отчётность**
   - Allure: шаги, attachments, метрики

6. **CI/CD**
   - `.gitlab-ci.yml`: стадии, артефакты, отчёты

## Ключевые месседжи для Senior‑уровня

- фреймворк масштабируемый и поддерживаемый
- тесты быстрые, стабильные, с контролем флейков
- CI/CD приносит прозрачность и качество

## Что подготовить заранее

- рабочий Allure отчёт
- 1–2 показательных теста (API + UI)
- короткий рассказ о trade‑offs
